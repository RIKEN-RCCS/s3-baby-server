# ETag機能

## 概要
ETagはリソースの内容を識別するための情報で、オブジェクトがサーバー内に配置されたときに算出しクライアントに値を返す。  
サーバーにアップロードされたファイルに対してETagを生成するアルゴリズムについて記載する。

## API
- CompleteMultipartUpload
- CopyObject
- DeleteObject
- GetObject
- HeadObject
- PutObject
- UploadPartCopy

## アルゴリズム概要
1. ファイル内容をハッシュ関数（MD5）を用いてハッシュ化する。
2. ハッシュ化された値を文字列としてクライアントに返す。

## 使用パッケージ
- github.com/aws/aws-sdk-go-v2

## 計算方法
アップロード元、アップロード先のファイル内容が同様の場合はETagも同じになる。
1. ファイル内容をバイト列に変換
2. バイト列をMD5関数に入力
3. 出力された16バイトの値をETag(`encoding/base64`)で文字列に変換して扱う

## 生成、保存タイミング
- サーバー上に新規作成されたオブジェクトに対しETagを算出しクライアントに返す。
- リクエストでETagを使うオプションが指定されたときに現在のサーバー上にあるファイルから算出し指定された値との比較を行う。
- マルチパートアップロードを使用してファイルがアップロードされたとき、本ファイルが作成されたときにMultiPartETagとしてメタデータへ記載し保存する。
